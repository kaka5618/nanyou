# 虚拟试衣落地页 V1 SPEC

## 一、Problem Statement（问题陈述）

当前普通消费者在购买服装前，往往只能依赖商品详情页的模特图、平铺图或他人买家秀来判断一件衣服是否适合自己。这种判断方式存在几个明显问题：

1. 商品图中的模特体型、肤色、气质与用户本人差异巨大，用户很难建立「这件衣服穿在我身上会是什么样」的直观认知。
2. 线下试衣需要专门去门店，时间成本高。
3. 即使在网上看中多件衣服，也无法快速验证哪件更适合自己或哪些可以搭配在一起。

**核心问题**：让普通消费者能够以最低门槛上传自己的照片和任意服装图片，在 **10～20 秒** 内获得一张可用于购买前参考的试穿预览图，从而降低购买决策犹豫，提高尝试意愿，并形成可持续反复使用的工具习惯。

**产品定位**：不是「精确量体」「尺码推荐」或「3D 虚拟上身」，而是 **低门槛、无需登录、以购买前参考为核心价值** 的轻量级虚拟试衣 Web 工具。

---

## 二、Proposed Solution（方案描述）

V1 将推出一个面向国内市场的独立 Web 落地页虚拟试衣工具。用户打开页面后 **无需注册登录**，依次上传一张自己的人物照片和一张服装图片，系统调用 AI 能力自动生成一张 **静态** 的试穿预览图。结果以「购买前参考图」的形式提供，而非精准尺码或真实上身承诺。

### 1. 核心用户

有线上购衣需求的普通消费者。包括但不限于：淘宝/天猫/京东/抖音电商平台的服装类目浏览者、对穿搭有兴趣但缺乏自信的年轻用户、希望快速验证多件衣服效果以减少退换货决策成本的人群。

### 2. 核心场景

以 **购买前预览** 为主，以穿搭尝试/娱乐分享为辅，同时兼顾一定程度的身材适配直观感受，但不把「精准版型判断」「尺码推荐」作为首版强承诺。

**典型场景举例**：

- 在某平台看到一件衣服 → 上传自己的照片 + 那件衣服的商品截图 → 获得预览参考。
- 手里有新衣服，想试试搭配某条裤子/裙子是否协调 → 分别上传后获得组合预览。
- 在社交平台上看到别人穿的某款衣服感兴趣 → 截图上传 + 自己的照片 → 快速判断是否适合自己。

### 3. 产品形态

- 独立 Web 落地页（Landing Page），以独立域名或路径对外提供服务；可作为常驻功能页运营，也可作为营销活动页投放使用。
- **不**嵌入商品详情页，**不**做 App 原生功能，**不**做小程序（后续迭代可扩展）。

### 4. 输入

用户一次完整试衣流程需要提供：

| 输入 | 说明 |
|------|------|
| 1 张人物照片 | 任意日常照片（自拍、他拍、半身、全身均可）；不强制标准站姿、正面、白底/纯色背景、高质量摄影 |
| 1 张服装图片 | 商品图、平铺图、挂拍图、随手拍、他人穿搭截图等；不强制白底抠图、平铺、单一角度 |

**唯一底线**：人物照片不能是仅有头部的特写（头像照），必须包含 **至少肩部以上的躯干部分**。通过事前引导（示例图 + 友好提示文案）实现，**不做硬性技术阻断**。

### 5. 支持品类

V1 支持：

- **上装**（T 恤、衬衫、外套、毛衣、卫衣等）
- **下装**（裤子、短裤、裙子等）
- **连衣裙 / 连体装**

一次试衣操作 **只替换一件衣服**；不支持整套同时替换。系统通过 AI 自动识别服装类别；V1 **暂不提供** 识别错误时的手动纠错机制；若结果不理想，用户可直接更换照片重新生成。

### 6. 识别与生成方式

- 系统自动识别服装类别和人体关键区域。
- V1 暂不提供识别错误时的专门兜底 UI（如手动框选区域等）；若结果不理想，用户可直接重新上传图片生成。
- 采用 **事前引导 + 分级友好提示** 的组合策略管理用户预期。

### 7. 输出

返回 **1 张静态** 试穿预览图。对外明确承诺为 **「AI 试穿预览图，仅供参考」**，不承诺：

- 精准尺码贴合度  
- 精准面料垂感与纹理还原  
- 精准真实上身一致性  
- 精确版型判断  

### 8. 生成目标（优先级）

| 优先级 | 目标 | 说明 |
|--------|------|------|
| P0 | 保持人物脸部与身份感不变 | 用户能一眼认出图中的人是自己 |
| P0 | 衣服类别正确 | 上装不会变成裤子；颜色和基本版型一致 |
| P0 | 结果对购买决策有参考价值 | 能辅助判断「适不适合我」 |
| P1 | 尽量保持原有身体轮廓与姿势不变 | 不出现肢体变形、比例失调 |
| P1 | 尽量保持原始背景不变 | 合成后背景与原图一致，不突兀 |
| P1 | 尽量还原服装关键特征 | 颜色、主要版型、长度、关键图案/印花 |
| P2 | 画面基本自然，不明显穿帮 | 无严重拼接痕迹、无逻辑错误（如两只左手） |

优先提供 **「购买前参考价值」**，而不是纯娱乐效果。

### 9. 复用机制

- **同一次页面访问（会话）** 中，人物图只需上传一次。
- 拿到第一张结果后，可连续更换服装图继续试穿，无需重复上传人像。
- 提供 **「换一件衣服」** 快捷入口，一键回到仅更换衣物图的流程。
- 人物图仅在 **当前浏览器会话** 中复用（前端状态保持），**不落库** 长期存储；**刷新页面后会话清空**，需重新上传。

**目的**：降低重复试衣的操作摩擦，最大化单次访问内的试衣次数。

### 10. 账号与历史记录

- V1 **不需要登录**，完全游客访问（使用量优先、降低漏斗流失、独立落地页流量珍贵）。
- **不提供** 跨设备/跨会话的历史记录。
- 当前会话内，前端可保留最近 **N 张（建议 3～5 张）** 合成结果供切换查看；**页面关闭即清除**，不做持久化。（V2 可考虑可选登录后保存历史。）

### 11. 商业模式与转化路径

V1 核心目标为 **使用量与传播性**，不以直接收费为主。

- **结果图水印**：右下角半透明品牌 Logo +「由 XX AI 试衣生成」。
- **分享**：带参数的唯一分享链接，落地回试衣页，裂变拉新。
- **免费无限用**：V1 不设次数限制、不设付费墙。
- **可选后续**（非 V1）：关联商品推荐/购买跳转、佣金、品牌合作、API 等。

## 三、Technical Constraints（技术约束）

### 核心框架

| 项目 | 方案 |
|------|------|
| 框架 | Next.js (App Router) 或纯 SPA，独立落地页 |
| 语言 | TypeScript |
| 运行时 | Node.js 服务端 API 调度 |

### 前端技术

| 项目 | 方案 |
|------|------|
| 框架 | React 19 / Vue 3（按团队偏好） |
| 样式 | Tailwind CSS |
| 组件库 | shadcn/ui (Radix UI) |
| 图标 | Lucide React |
| 状态 | React Context API / Zustand |

### AI 集成

| 项目 | 方案 |
|------|------|
| 能力来源 | 国内大厂云服务 API（阿里云 / 腾讯云 / 百度云等虚拟试衣或图像生成相关 API） |
| 调用方式 | **后端代理**，API Key 不暴露前端 |
| 备选 | 同时对接 2～3 家做 A/B，上线前选定最优 |
| 内容安全 | 同一厂商或专业审核 API，过滤涉黄、涉暴、政治敏感等 |

### 基础设施

| 项目 | 方案 |
|------|------|
| 图片存储 | 对象存储（OSS / COS 等）；原图与合成结果均转存 |
| 图片处理 | 前端上传前客户端压缩：分辨率上限 2048px，质量 0.8，单张不超过 2MB |
| CDN | 静态资源 + 图片走 CDN；首屏加载 ≤ 2 秒（4G） |
| 部署 | 独立域名，全站 HTTPS |
| 包管理器 | pnpm |

### 开发规范

- 代码规范：Airbnb Style Guide  
- Git：Feature Branch，PR Review 后合并  
- 浏览器：iOS Safari 14+、Android Chrome 80+、微信内置浏览器、PC Chrome/Edge/Safari 最新两版本

## 四、Non-goals（明确不做的事）

| # | 不做 | 说明 |
|---|------|------|
| 1 | 尺码推荐 | 不输出 S/M/L 或具体尺码 |
| 2 | 精确身材测量 | 不输出肩宽、腰围等量化数据 |
| 3 | 复杂场景保证效果 | 多人合照、严重遮挡、极端角度、坐蹲姿、极模糊/极暗、仅头部特写等 → 友好提示换图 |
| 4 | 自动穿搭推荐 | 不自动推荐搭配单品 |
| 5 | 整套一次替换 | 一次仅一件：上装、下装或连衣裙之一 |
| 6 | 动态试衣 | 仅静态图，无视频/GIF |
| 7 | 账号登录体系 | V1 完全免登录 |
| 8 | 历史记录持久化 | 无「我的试衣记录」等 |
| 9 | 高风险人物专项风控 | 仅通用内容安全；不做未成年人/公众人物专项 |
| 10 | 高精度真实性承诺 | 文案统一「AI 预览，仅供参考」 |
| 11 | 购买跳转/商品关联 | V1 结果页不挂「去购买」、无商品列表 |
| 12 | PC 端优先 | **移动端优先**；PC 基础可用即可 |

## 五、Success Criteria（成功标准）

### 1. 单次结果成功标准

一张合格的试穿预览图应 **同时** 基本满足：

1. 用户能一眼认出是自己（面部、发型、体态辨识度）。  
2. 衣服类别正确（传入 T 恤则生成 T 恤等）。  
3. 对购买决策有参考价值（能形成「大概适合/不太适合/可以一试」的判断）。  
4. 画面基本自然，无明显穿帮、肢体逻辑错误、严重 AI 伪影。  

### 2. 产品层面

| 指标 | 目标 |
|------|------|
| 首次试衣完成率 | ≥ 40%（访问 → 完成首次试衣） |
| 单次访问平均试衣次数 | ≥ 1.8 次 |
| 「换一件衣服」点击率 | ≥ 30% |
| 7 日回访率 | ≥ 15% |

### 3. 业务层面（早期信号）

- 分享意愿：分享率 ≥ 15%  
- 品牌曝光：带水印图外部传播量增长  
- 新客来源：分享链接进入占比 ≥ 30%  
- UGC：非诱导的自发提及  

### 4. 不作为 V1 核心指标

尺码准确率、身材测量准确率、专项风控识别率、动态试衣、全品类覆盖、直接 GMV、注册转化率等。

## 六、一句话版本

面向国内普通消费者的独立 Web 落地页虚拟试衣工具：用户 **无需登录**，上传自己的照片和任意服装图片，系统调用国内大厂 AI 在 **10～20 秒** 内生成 **静态** 试穿预览图；以 **购买前参考** 为核心价值，以 **低门槛、会话内复用、分享裂变** 为策略；**不做** 尺码推荐、账号体系、高精度真实上身承诺；**核心 KPI** 为使用量与重复使用率。
